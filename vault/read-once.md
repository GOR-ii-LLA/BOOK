# read-once
Категория: Claude
Официальная страница: 
GitHub: https://github.com/Bande-a-Bonnot/Boucle-framework/tree/main/tools/read-once
Подробнее: read-once — это хук для Claude Code, который следит за тем, какие файлы уже были прочитаны за сессию, и блокирует их повторное чтение, если файл не изменился.
Представь: ты работаешь с большим проектом, и Claude Code снова и снова перечитывает один и тот же файл с настройками — каждый раз тратя токены впустую; read-once замечает это и говорит «стоп, этот файл уже в памяти».
По данным автора, хук экономит 30–40% токенов на чтение файлов за сессию, а в длинных автономных сессиях — до 60–90%; есть режим diff, который при изменении файла отправляет не весь файл заново, а только изменённые строки (экономия 80–95% на правках).
Удобная интеграция с Claude Code: устанавливается одной командой curl, подключается как PreToolUse-хук и работает полностью прозрачно — Claude продолжает работать, просто видит предупреждение вместо лишнего чтения.
Важный нюанс: по умолчанию хук работает в режиме warn (предупреждает, но не блокирует) — это нужно, потому что инструмент Edit требует предшествующего Read; режим deny даёт максимальную экономию, но ломает Edit, используй его осторожно.
